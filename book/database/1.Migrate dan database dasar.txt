
untuk membuat database pada laravel 11 bisa dari IDE nya langusng cara nya adalah 

1. Periksan koneksi di ENV
-buka File beruliskan ".env" kemudian klik cari bagian ini


DB_CONNECTION=sqlite
# DB_HOST=127.0.0.1
# DB_PORT=3306
# DB_DATABASE=laravel
# DB_USERNAME=root
# DB_PASSWORD=


biasa nya ada di baris ke 21 ke atas kemudian konfigurasi sesuai datbase mana yang ingin di pakai
jika menggunakan Mysql maka ganti ke konfigurasi berikut

DB_CONNECTION=Mysql  //ini nama CONNECTION dari database yang di gunakan
DB_HOST=127.0.0.1    //ini nama host yang di gunakan biasa nya 127.x.x.x. atau localhost
DB_PORT=3306   //port nya di sini
DB_DATABASE=laravel   //nama database yang di gunakan untuk menyimpan data
DB_USERNAME=root  //username database
DB_PASSWORD=   //password bila ada masukan di sini jika tidak ada kosongkan

2.jika koneksi sudah di pastikan terhubung maka kita bisa buat sebuah table baru tapi sebelum nya 
  buka folder  >bernama database di dalam nya ada 2 table migraten dan factories klik pada folder > migrate
  di dalam migrate ada 3 file bawaan dari laravel yaitu

 > 0001_01_01_000000_create_users_table.php
 > 0001_01_01_000001_create_cache_table.php
 > 0001_01_01_000002_create_jobs_table.php

nah ke tiga file itu sebenarnya ada table pada bawaan dari laravel jadi bukan file sembarangan
nah sudah di ketahui di dalam folder migrate ada table kemudian kita bisa buat sebuah table khsus sesuai yang kita masukan
pada cmd yang aktif di project ini atau bisa juga klik crtl + ` ketik pada perintah

php artisan make:migration create_NamaTable_table --create=NamaTable

kemudian enter maka dengan otomatis sebuah file atau table baru akan tertambah di folder migrate jika seperti printah di atas maka
di file akan terlihat seperti ini

2025_09_09_064022_create_NamaTable_table 

yang di kiri
2025_09_09_064022

itu adalah tanggal kapan di buat dan terakhir di update atau di migrate kembali

yang di kanan
create_NamaTable_table

itu adalah nama table yang tadi sudah di buat pada cmd

3.mengirim migration ke dalam database 

nah sebenarnya kita baru membuat sebuah skema nya saja bentuk fisik dari database nya belum ada ibarat kan 
kita baru membuat Query nya saja belum di eksekusi dan di buat kan database nya cara untuk mengirim nya ke database
adalah

buka kembali cmd nya > ctrl + ` 
ketik pada cmd 

php artisan migrate 

kemudian enter mah jika sudah maka CMD akan menampilkan sebuah info seperti ini

  0001_01_01_000000_create_users_table ................................................................................................ 62.92ms DONE
  0001_01_01_000001_create_cache_table ................................................................................................. 7.19ms DONE
  0001_01_01_000002_create_jobs_table ................................................................................................. 80.69ms DONE


ciri nya adalah tanda done atau succes jika sebalik nya malah menampilkan sebuah eror berarti perintah sedang bingung 
apa yang harus dia kirim kan ke database nah sebagai tanda unuk memberitau apa yang di kirim ke database yaitu dnegan perintah

php artisan  migrate:fresh

maka data akan
